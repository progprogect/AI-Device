# Дорожная карта ESP Sense

## Ключевые решения

- **Фреймворк: Flutter** + нативный `whisper.cpp` через FFI. BLE — `flutter_blue_plus`,
  UI и HTTP-видео кроссплатформенны; тяжёлый ML — одна C++ библиотека для iOS и Android.
- **Модель Whisper: `large-v3-turbo` Q5_0 (~574 МБ)** — точность уровня large,
  быстрый декодер, на iPhone 14/15 работает почти в реальном времени.
  Фолбэк `small` в настройках. Модель скачивается при первом запуске.
- **Потоковая расшифровка**: псевдо-стриминг (буфер → VAD → окна 3–8 с → whisper
  каждые ~2 с с контекстом). Задержка ~1–3 с.
- **Установка iOS**: сборка IPA на Mac → Sideloadly/AltStore с бесплатным Apple ID
  (переподпись раз в 7 дней). Android: APK через Telegram.
- **Транспорты**: BLE — команды/аудио/фото; Wi-Fi — видео MJPEG.

## Этапы

### Этап 1 — Документация ✅
- `docs/ARCHITECTURE.md` — слои, GATT-спецификация, диаграммы последовательности.
- `docs/PLAN.md` — этот файл.
- `docs/INSTALL.md` — установка на iOS (Sideloadly) и Android (Telegram APK).

### Этап 2 — Прошивка v2 (`firmware/esp_sense_ctrl/`)
- Командная модель вместо «всегда стримит»:
  - `commands` (WRITE): start/stop audio, capture photo, set wifi;
  - `data` (NOTIFY): аудио и фото-чанки (формат прежний);
  - `device_info` (READ): версия и возможности.
- Компиляция с `PSRAM=opi`, прошивка по USB, проверка Python-скриптом.

### Этап 3 — Flutter-приложение (`app/`)
- Экран подключения: скан BLE, статус, разрешения.
- Экран управления: Start/Stop аудио, Photo, Start/Stop видео, последний снимок,
  лента расшифровки (частичный + финальный текст).
- Модули: `ble_service.dart`, `audio_pipeline.dart` (буфер + VAD),
  `whisper_service.dart` (FFI), `model_downloader.dart` (загрузка + SHA-256).

### Этап 4 — Сборка и установка
- iOS: `flutter build ipa --no-codesign` → Sideloadly → iPhone.
- Android: `flutter build apk` → файл в Telegram → установка.

## Обновления без переустановки тяжёлых компонентов

1. **Модель отдельно от приложения** — качается один раз в Application Support /
   internal files; переустановка IPA с тем же Bundle ID сохраняет данные.
   Манифест `models.json` (версия + SHA-256) — докачка только при изменении.
2. **Shorebird code push** — патчи Dart-кода без переустановки приложения.
3. **`CMD_OTA_BEGIN`** — зарезервировано для OTA прошивки ESP.

Важно (iOS): модель переживает обновление, но не удаление приложения.
Если подпись истекла — ставить новую версию поверх, не удаляя старую.

## Будущие доработки

- Кнопки на устройстве → `PKT_EVENT` → тот же pipeline в приложении.
- Обратный канал: команды к устройству (LED, звук, TTS).
- Обработка расшифрованного текста (LLM, команды голосом).
- Облачный relay для работы вне одной сети.
- OTA прошивки ESP по BLE/Wi-Fi.

## Риски

- **BLE-полоса**: PCM 16 кГц ≈ 256 кбит/с — на пределе. План Б: 8 кГц или ADPCM.
- **7-дневная переподпись** free Apple ID — неизбежно без платного аккаунта ($99/год).
- **large-v3-turbo на iPhone 14 (A15)**: если медленно — авто-фолбэк на `small`.
